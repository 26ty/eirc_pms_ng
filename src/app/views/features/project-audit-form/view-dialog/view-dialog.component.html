職責(Responsibility):
專案負責人在此專案職責中，針對關鍵里程碑時程、專案成本及範圍條件等，需符合既定計畫實施，同時遵守適
當的品質管理標準。專案負責人須與功能性經理保持聯絡，以確保所有被分配的資源能有效地和高效率地被使用
及被完全接受。
另外，專案負責人需負起以下職責：
  1. 在團隊成員、顧客、供應商、公司管理階層之間，建立及維持必要的溝通管道。
  2. 設定實際的專案計畫，並且能得到客户及供應商的認同及接受。
  3. 建立和維護專業組織，授予成員明確的角色及權責，並參與專案團隊成員的續效衡量。
  4. 擬訂專案計畫，與專案成員規劃所有的專案細節，設定專案里程碑及查核點，並獲得發起人的認同及接受。
  5. 管理及控制專業成本、時程、技術範園、人力及物料資源及品質。
  6. 即時發掘品質、規格、成本或進度的潛在問題，並執行矯正。
  7. 持續利用週報告(細節)、月報告(綱要)等，以向上或對下說明管理情形及進行協調溝通。
  8. 將成本及成果，以實際成效與預定成效做比較，必要時，需即時採取合適的解決方式。
  9. 隨時維護專案計畫，以使其能持續地呈現實際時程、費用、及由功能性經理所允諾的資源表現情形。
職權(Authority):
為確保專案能符合原訂的宗旨，專案負責人被授權依據公司的專案管理政策及規章來管理專案及發佈指示。
另外，專案負責人同時具有以下權限：
  1. 直接與顧客聯繫所有與專案有關的事宜。
  2. 直接執行與專案有關的事宜。
  3. 必要時可更新專案計畫，同時亦需得到發起人允許。
  4. 可向專案成員們或其他專案關係人提出週期性狀況報告之要求。
  5. 可要求相關功能性部門協助提供與此專案有關的資源。
  6. 為使能符合專案需求，必要時不必局限於呈報流程，可與所有管理階層進行協調。
  7. 可向功能性部門提出指派/調換成員之要求。
  8. 經功能性部門主管所同意指派的人員，必須盡全力協助專案負責人以達成專案目標。
